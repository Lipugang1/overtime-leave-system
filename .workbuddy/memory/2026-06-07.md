### Turbopack 中文路径崩溃
- Next.js 16.1.1 的 Turbopack 在中文路径下会 panic
- 错误日志：`reading file E:\扣子调休管理系统\nul` → Windows 保留设备名导致 `os error 1`

- `--no-turbopack` 命令行参数在 Next.js 16 中不存在
- **解决方案**：使用 `next build && next start` 生产模式启动（跳过 Turbopack dev server）

### 密码验证格式不匹配
- 数据库中 admin 密码是 **bcrypt** 格式（`$2b$10$...`）
- `src/lib/password.ts` 的 `verifyPassword()` 只支持 **PBKDF2** 格式（`iterations.salt.hash`）
- bcrypt hash 用 `.` 分割后只有 2 段，`parts.length !== 3` → 返回 false
- **修复**：`verifyPassword` 现在同时支持 bcrypt 和 PBKDF2 格式

**根因**：
1. `supabase-client.ts` 的 `loadEnv()` 只调用 `dotenv.config()`，默认加载 `.env` 而非 `.env.local`

## JWT 密钥一致性修复

### 问题：登录成功但其他 API 返回"登录已过期"

**根因**：EdgeOne SSR 为每个 API 路由启动独立的 Node.js 函数实例，各实例的 `process.env` 独立。
`auth-shared.ts` 中 `JWT_SECRET = process.env.JWT_SECRET || default` 在不同实例中取值不一致：
- 某些实例加载了 `.env.local` → `JWT_SECRET = 'overtime-system-secret-key-2024'`
- 某些实例未加载 → `JWT_SECRET = 'overtime-management-secret-key-2024'`
- 登录用密钥 A 签名，其他路由用密钥 B 验证 → 验证失败

**修复**：`JWT_SECRET` 改为只使用硬编码的 `JWT_FALLBACK_SECRET`，不再依赖 `process.env`
```typescript
const JWT_FALLBACK_SECRET = 'overtime-system-jwt-secret-key-fixed';
export const JWT_SECRET = JWT_FALLBACK_SECRET;
```
